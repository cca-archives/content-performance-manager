GOV.UK Content Performance API v1.0.0

GOV.UK Content Performance API provides metrics about GOV.UK content.

This API accepts HTTP requests and responds with JSON.

Base URL

All requests to this API should be prefixed with the following URL:

https://content-performance-manager.publishing.service.gov.uk/api/

Authentication

  • Bearer authentication (header format: Authorization: Bearer <your-token-here>)

Metadata

GET /metrics/

List all available metrics.

Returns a list of metrics that the API can return.

This request requires authentication.

Example Request

curl 'https://content-performance-manager.publishing.service.gov.uk/api/v1/metrics/'

Example Response

Examples

[
  {
    "description": "Number of pageviews",
    "source": "Google Analytics",
    "metric_id": "pageviews"
  },
  {
    "description": "Number of unique pageviews",
    "source": "Google Analytics",
    "metric_id": "unique_pageviews"
  },
  {
    "description": "Number of reported content issues",
    "source": "Feedback explorer",
    "metric_id": "feedex_comments"
  },
  {
    "description": "Number of .pdf attachments",
    "source": "Publishing API",
    "metric_id": "number_of_pdfs"
  }
]

Responses

Status Meaning Description Schema
200 OK Success response MetricsResponse

Aggregations

GET /metrics/{metric}/{content_id}

Get summarised metrics for a content item.

Returns a summary of a metric based on data collected over a time period.

This request requires authentication.

Parameters

Parameter In Type Required Description
content_id path string true A GOV.UK content ID. This is a UUID value as used in the Content API.
metric path string true The name of a metric.
from query date true From date in the format ‘YYYY-MM-DD’. Only data collected on or after this date will be included.
to query date true To date in the format 'YYYY-MM-DD’. Only data collected on or before this date will be included.

Example Request

curl 'https://content-performance-manager.publishing.service.gov.uk/api/v1/metrics/pageviews/f838c22a-b2aa-49be-bd95-153f593293a3/summary?from=2018-02-27&to=2018-02-27'

Example Response

Examples

{
  "property1": {
    "total": 0,
    "latest": 0
  },
  "property2": {
    "total": 0,
    "latest": 0
  }
}

{
  "title": "One or more parameters is invalid",
  "invalid_params": {
    "from": [
      "can't be blank",
      "Dates should use the format YYYY-MM-DD"
    ],
    "to": [
      "can't be blank",
      "Dates should use the format YYYY-MM-DD"
    ],
    "content_id": [
      "Content ID must be a UUID."
    ]
  },
  "type": "https://content-performance-api.publishing.service.gov.uk/errors/#validation-error",
  "x-old-ref": "#/components/examples/ErrorExample"
}

Responses

Status Meaning Description Schema
200 OK Success response SummaryResponse
400 Bad Request The API request is invalid. ErrorResponse

Time series

GET /metrics/{metric}/{content_id}/time-series

Get metric values for a single content item over a date range.

Returns an array of values for each metric. Each item represents a metric value at a single point in time.

This request requires authentication.

Parameters

Parameter In Type Required Description
content_id path string true A GOV.UK content ID. This is a UUID value as used in the Content API.
metric path string true The name of a metric.
from query date true From date in the format 'YYYY-MM-DD’. Only data collected on or after this date will be returned.
to query date true To date in the format 'YYYY-MM-DD’. Only data collected on or before this date will be returned.

Example Request

curl 'https://content-performance-manager.publishing.service.gov.uk/api/v1/metrics/f838c22a-b2aa-49be-bd95-153f593293a3/time-series?from=2018-02-27&to=2018-02-27&metric=pageviews'

Example Response

Examples

  "property1": [
    {
      "date": {},
      "value": 0
    }
  ],
  "property2": [
    {
      "date": {},
      "value": 0
    }
  ]
}

{
  "title": "One or more parameters is invalid",
  "invalid_params": {
    "from": [
      "can't be blank",
      "Dates should use the format YYYY-MM-DD"
    ],
    "to": [
      "can't be blank",
      "Dates should use the format YYYY-MM-DD"
    ],
    "content_id": [
      "Content ID must be a UUID."
    ]
  },
  "type": "https://content-performance-api.publishing.service.gov.uk/errors/#validation-error",
  "x-old-ref": "#/components/examples/ErrorExample"
}

Responses

Status Meaning Description Schema

Objects

Metric

Sample JSON

{
  "metric_id": "string",
  "description": "string",
  "source": "string"
}

Fields

Name Type Required Description
metric_id string true ID to refer to the metric within the API
description string true Description of the metric
source string false Where this metric comes from

Summary

Sample JSON

{
  "total": 0,
  "latest": 0
}

Fields

Name Type Required Description
total integer true Sum of all recorded metric values within the date range.
latest integer true The latest recorded metric value within the date range.

Name Type Required Description
